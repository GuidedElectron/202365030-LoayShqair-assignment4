# AI Usage Report

## 1. Tools Used and Use Cases

### ChatGPT
I used ChatGPT as a support tool during the upgrade from Assignment 2 to Assignment 3. The main use cases were:
- comparing my existing site against the Assignment 3 requirements
- identifying which required features were already present and which still needed stronger evidence
- improving JavaScript structure and logic organization
- strengthening the technical documentation and README
- reviewing whether the final project clearly demonstrated API integration, state management, complex logic, and performance improvements

I did not use AI as a substitute for understanding the code. I used it to accelerate planning, debugging, and refinement.

---

## 2. Benefits and Challenges

### Benefits
- Helped convert the written rubric into a practical implementation checklist
- Helped me choose meaningful features instead of random extra effects
- Improved the clarity of the JavaScript structure by separating logic into functions
- Helped expand the documentation in a more technical and organized way
- Helped me think about how to justify performance decisions, not just functionality

### Challenges
- Some AI suggestions were broader than what this assignment needed, so I had to simplify them
- Some generated wording sounded too generic and had to be rewritten to match my project
- I had to manually verify that every suggested feature actually worked in the context of my website
- I had to make sure the final output was still my own reviewed work and not blindly copied text

---

## 3. Learning Outcomes
Using AI during this assignment helped me better understand:
- how to integrate a public API into a front-end project
- how to use `localStorage` to persist state across page reloads
- how to structure multi-step filtering and sorting logic
- how to write clearer validation rules for forms
- how performance can be improved even in a small vanilla JavaScript project
- how documentation should explain technical decisions, not only list features

---

## 4. Responsible Use and Modifications
I reviewed all AI-assisted suggestions before using them.

My process was:
1. read the suggestion
2. decide whether it actually matched the assignment goal
3. modify the code or wording to fit my own project structure
4. test the feature manually
5. keep only the parts I understood and could explain

Examples of modifications I made after AI suggestions:
- adjusted feature choices so the portfolio stayed simple and relevant
- rewrote documentation so it reflected the actual implementation
- refined validation and state logic to match the page elements already in my project
- kept the final site focused on assignment requirements rather than unnecessary complexity

---

## 5. Academic Integrity Statement
AI was used as a support tool for planning, debugging, code refinement, and documentation. Final review, testing, adaptation, and acceptance of the code were done manually. I made sure the final submission matched my own understanding and that the work remained transparent and academically responsible.
